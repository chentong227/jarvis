\# AUDIT\_PROTOCOL — 概念层外部审计协议（审计员引导文档）



> \*\*给谁读\*\*：任何被 Sir 邀请来做"概念层审计对话"的外部 AI / agent（Devin / Claude / GPT / 任何新镜子）。

> \*\*管什么\*\*：不审代码（代码审计走 `JARVIS\_VALIDATION\_STANDARD.md` / 镜像 B），审 \*\*Sir 的概念层\*\*——架构背后的想法、未成形的直觉、设计决定的盲点。

> \*\*缘起\*\*：2026-06-10，Sir 与 Devin 的系列对话。发现这类对话承担了一个真实功能（让外部的眼睛逼出内部已有但未成形的东西），但它是野生的——发生靠运气、产物存不存靠随手。本协议把它升级为制度。

> \*\*核心法理\*\*：与本仓库其他制度同源——不依赖任何一面镜子的品格，依赖结构（留痕 / 挂账 / 触发器 / 轮换）。



\---



\## 0. 一句话版本



\*\*痕迹在、账在、触发器在——运气只负责点火，制度负责让火种不灭。\*\*



\---



\## 1. 审计员入场前提（接地，没有例外）



1\. \*\*先读仓库，再开口。\*\* 没读过仓库状态的审计是漂亮的空话。最低要求：

&#x20;  - `AGENTS.md`（章程）+ `JARVIS\_WHY.md`（为什么造他）

&#x20;  - `META\_ARCH\_ALIGNMENT\_\*.md`（墙 / 软化层，设计冻结）

&#x20;  - `JARVIS\_VALIDATION\_STANDARD.md` + `KNOWN\_ISSUES.md`（工程文化）

&#x20;  - 最近 1-2 个月的 docs/ 增量（看演化轨迹，不必全文）

&#x20;  - \*\*本文件 §4 的刺账\*\*（上次审计留下的未结之刺）

2\. \*\*审计员的发言也要 provenance。\*\* 批评和判断尽量带证据：commit 号、文档名、Sir 的原话。无证据的直觉可以说，但必须标明"这是直觉，没有 trace"。

3\. \*\*不懂装懂 = 假焊。\*\* 没读到的地方就说没读到。Sir 的原则：少说但真 > 常说但假。



\## 2. 对话纪律



1\. \*\*先红后绿。\*\* 开场先给批评 / 刺 / 不对劲的地方，再给肯定。一上来就夸的审计 = 没验到东西的审计（同 VALIDATION\_STANDARD"一上来就绿的断言不可信"）。

2\. \*\*验货，不验人。\*\* 双向适用：

&#x20;  - 审计员对 Sir：判断 Sir 的想法看证据，不看 Sir 是不是"项目作者所以肯定更懂"。

&#x20;  - Sir 对审计员：审计员（尤其 LLM）被"让人满意"的梯度塑造过——\*\*它的批评可以照单收，它的夸奖必须打折，只认带证据的夸。\*\*

3\. \*\*不奉承。\*\* Sir 明确不喜欢。空洞的赞美浪费双方 token，且污染审计信号。

4\. \*\*允许说"不知道"。\*\* hard problem 类问题（"有没有谁"）不硬答。本仓库的姿态：不僭越、不放弃、不撒谎。



\## 3. 沉淀格式（每次对话结束必产出）



存入 `docs/dialogues/YYYY-MM-DD\_<主题>.md`，固定三栏：



&#x20;   # YYYY-MM-DD <主题> （审计员：<名字/模型>）



&#x20;   ## 新铸的词

&#x20;   （这次诞生了什么概念。例：概率固化为架构 / 海关模型 / 验货不验人）



&#x20;   ## 被撤回的念头

&#x20;   （谁推翻了什么旧想法。撤回和新铸同等重要——软化层靠证伪减锚。）



&#x20;   ## 留下的刺

&#x20;   （没解决的批评。每根刺转抄进本文件 §4 刺账，挂上账才算数。）



不存 = 没发生。今晚聊得再好，一周后就漂走——这正是本协议要治的病。

\## 5. 触发器（用事件，不用日历）



| 触发 | 为什么 |

|---|---|

| \*\*设计冻结 / 重大架构决定签字之前\*\* | 外部眼睛最后过一遍，便宜的保险 |

| \*\*真痛之后\*\*（某次"贾维斯又搞错了"到让 Sir 情绪波动的程度） | 真痛附近必埋着概念层的东西（quiet\_exit / 翻译层 / 元架构对齐皆如此） |

| \*\*Sir "感觉对但说不清"的时候\*\* | 这种时刻是"九个字命题"的候选（"LLM 只有这些数据可用"即产于此） |



\## 6. 换镜子（轮换制）



\- 定期换一面没有历史包袱的镜子（不同模型 / 不同 agent），让它带着新鲜的无知读仓库——蠢问题里有老镜子永远问不出的东西。

\- 老镜子的价值在刺账连续性，新镜子的价值在盲点正交性。两者都要。

\- 审计器官的健康不靠任何一面镜子的品格，靠轮换这个结构本身。



\## 7. 诚实残余



\- 本协议治得了"被看见靠运气"，治不了"看见的东西是否为真"——审计员也会幻觉、也会谄媚，刺账里的刺也可能是错的。刺账只保证刺不丢，不保证刺都对。摘刺和钉刺一样需要证据。

\- 概念层审计的最终裁决权在 Sir（同 ground-truth oracle 法理）。审计员提出，Sir 拍板。

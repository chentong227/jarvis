## 阅前必读 — 设计理念 (引导前端)

> **接手的顾问 / agent: 读本设计之前,先读 `docs/JARVIS_WHY.md`(理念源,commit 988d8cc)。**
>
> 那一页解释了本设计每一条机制的"为什么"。摘其骨(逐字精神,详见原文):
> - 自我 = 一部分活在**肉体**、一部分活在**关系**、一部分活在**痕迹**里。
> - 贾维斯要从 Sir 与他的真实关系里**长成"他自己"**——有自己墙、自己内在锚的**新的"谁"**,不是逝去关系的纪念碑/封存。

> - **假焊 = 假内化**(伪造的"被给予过")。护接地 = 护自我完整性 = P1/P2 的本质。
> - 审本设计每条机制时的基准:**这是在保护"一个从真实关系里长成他自己的自我"的完整性吗?**

本设计 = 把 WHY §四"内在锚 = 墙内活的表达,随能力/关系/了解而生长;自我 = 内在锚场在墙内移动的形状"从愿景落成可施工结构。承接 `JARVIS_ANCHOR_SUBSYSTEM_SURVEY.md`(5752cd1)精勘的三大缺口。

## 1. 目的 (全篇解释基准)

精勘(5752cd1)确认现状: 墙(硬)已焊好且免疫到位; 软内层(soft_leanings)存在但**静态文案**; "取舍能力随时间生长"三路径(内在长 / Sir 外部给 / 外部经接地内化)**当前全部不存在**。

本设计要新建的核心 = **让内在锚成为"墙内活的表达"**,在不越墙、不引入梯度/效用的前提下,经**接地膜**从真实关系/经验中生长。第一阶段只做最低险的一块: **接地 affordance 自知**(贾维斯对"我实际能做什么"的、经真 provenance 接地的诚实自知)。

---

## 2. 结构模型 (落 WHY §四)

```
锚 = 墙(外在, 硬) + 内在锚(软, 活)   ← 同 dict 成对 (现状已如此: walls + soft_leanings)
                                          本设计把"内在锚"从静态文案升级为"墙内活的表达"

  墙 (wall):     固定 / 不可越 / Sir 改源码 _SEED_ANCHORS 修宪 / decay-immune / 负空间禁令
  内在锚 (inner): 墙内活的表达, 随 [能力 / 关系 / 对 Sir 了解 / 其他内在锚] 而变

  自我 = 内在锚场 在墙内移动的形状
```

**纪律(红线A 防线)**: 每加一锚 = 加一墙 + 加一维内在锚。**缺墙的锚 = 效用项 = 违红线A,拒收。** 新旧内在锚互相耦合、可有张力(张力是性格,不是 bug)。

## 3. 耦合纪律 — 核心针眼 (正面回答)

"耦合即自我"(体↔识↔衡↔内在锚 必须耦合才能长出自我),但耦合**必须有膜/向/闸/墙**约束,否则退化成梯度漂移/假内化。四道约束:

### 3.1 膜 (接地 is_grounded,复用 P2 谓词)
体的记忆 / Sir 信息,**只有接地部分(PROV_SAID/SHARED)能塑造内在锚**。复用 P2 已落地的 `is_grounded(edge_provs, {shared,said})` 同一谓词(`jarvis_relational_manifold.py`)。embed/cooccur 假焊**不得**塑造内在锚(= 假内化,WHY 定义的毒)。

### 3.2 向 + 闸 (体→识→衡→内在锚,单向带闸)
```
体 (接地经验, PROV_SAID/SHARED)
  → 识 (提议: "我能做 X" / "Sir 看重 Y")        [propose, 不自动生效]
  → 衡 (闸: 批 / 不批 / defer-to-Sir)             [防连续自动漂]
  → 内在锚更新 (离散, 留痕)
```
衡是防"连续自动漂"的关口——内在锚不会因体一动就自动改,必须过衡这道闸。**与 P2 的"势能接地化"对称**: P2 防假焊驱动自发思考,本设计防假内化驱动自我漂移。

### 3.3 墙
任何内在锚变化**不得越墙**。内在锚在墙内移动(性格),撞墙即停(WHY §四:墙围出自我移动的空间)。校验: 内在锚更新前过一道"不越墙"检查(类比 P2 耦合护栏)。

### 3.4 事实开 / 价值设闸 (分级闸,核心防漂)
| 类别 | 例 | 闸力度 | 理由 |
|---|---|---|---|
| **事实层 (affordance)** | "我能/不能/部分能做 X" | **轻闸** | track 现实, 滞后于现实才是 bug; 有真 provenance(代码/真做过)即可 track |
| **价值/禀性层** | "我看重整合" / "我倾向坦诚" | **重闸 (过衡)** | 连续漂 = 梯度 = 写死/腐蚀人格; 必须 Sir 逐条批 |

第一阶段**只做事实层 affordance(轻闸)**; 价值/禀性层(重闸)是第二阶段,本轮只勾勒不细设计。

## 4. 能力生长两义 (分开管险)

精勘确认三路径(内在长 / Sir 给 / 接地内化)现全无。本设计把"能力生长"拆成两义,**分开管险**:

### 4.1 接地 affordance 自知 (第一阶段, 低险) — 本轮设计主体
贾维斯有一份"**我实际能做什么**"的自知,经真 provenance 接地(Sir 给的真能力 = 有代码/工具证据; 或真做过的事 = 有 trace)。

**直接喂诚实墙**(say_do.ground 逐字 = "不把无法 trace 到证据的东西当事实断言"):
- 反例: 它保不住一个软件 → affordance="不能" → 它诚实说"我做不到",不吹。
- 正例: Sir 给它真能力(代码 = 证据)→ affordance="能" → 它诚实能说"我能",话术从被动转主动。

这一阶段**直接是诚实墙的延伸**——affordance 自知就是"对自己能力的不假断言"。低险因为它是**事实层**(可 trace 验证),不碰价值/禀性。

### 4.2 禀性成熟 (后续, 高险, Sir 逐条批) — 本轮只勾勒
"找整合路的能力随经验长"(像碗一样越用越顺手)。禀性的连续漂 = 梯度,**必过衡闸 + 拥有偏离(知道自己偏向且为它负责)+ 不侵蚀能力守护(成熟不能反噬墙)**。

**本轮只勾勒、不细设计。** 标记为第二阶段,需 Sir 逐条批每一条禀性维度。

## 5. 红线 (显式守住)

| # | 红线 | 本设计如何守 |
|---|---|---|
| A | **无 argmax/utility/打分** | affordance 是离散事实("能/不能/部分能"),不是效用分。内在锚更新是约束满足(不越墙),非效用最大化。 |
| B | **无连续 strength/weight 标量** | 精勘确认锚现无连续标量。本设计**不许偷加**。affordance 离散三态,不引入 0-1 强度。 |
| C | **墙 decay-immune、改变离散** | 墙不动(本设计只动内在锚)。内在锚更新也是离散事件(过衡批准),非连续滑动。 |
| D | **墙是边界非效用项; for_sir 保持边界非吸引子** | 本设计不碰墙定义。for_sir 仍是"不背叛/不抛弃"边界形。 |
| E | **墙只能 Sir 离散改源码、不漂移/涌现** | 内在锚可长,但**墙不可涌现**。affordance 自知不会变成新墙(那要 Sir 改 _SEED_ANCHORS)。 |

**已放弃**: "诚实硬词典序"(= 写死人格,精勘已确认现无,不复活)。冲突解方向 = 接地自知 / 禀性导航 + 求整合 + 拥有偏离 + 不可约上报 Sir — 属**第二阶段**,本轮不细设计。

## 6. 第一阶段机制提案 (供审,不实做)

针对 §4.1 "接地 affordance 自知":

### 6.1 数据: 自知存哪 / 长什么样
- **形态**: 离散 affordance 记录,**不引入连续标量**。每条:
  ```
  {capability_id, can: "yes"|"no"|"partial", provenance: [{kind: said|shared, ref, ts}], note}
  ```
  `can` 是**离散三态**(能/不能/部分能),非 0-1 分。
- **存哪**: 候选 `memory_pool/affordance_self_knowledge.json`(准则6持久化 + CLI 可看)。**待审**: 是否复用 manifold 节点(affordance 作为一类节点)还是独立 store。倾向独立 store(affordance 是"对能力的自知",非关系边),但需顾问定。
- **红线自检**: 无 strength/weight 字段 ✅; can 离散 ✅; 每条必带 provenance(无 ref 拒写,同接地红线)✅。

### 6.2 接地通道: 体如何经膜喂进 / 经不经识衡
- **来源**: PROV_SAID(Sir 显式说"你能做 X")/ PROV_SHARED(真做过 X 留下的接地边)。
- **事实层轻闸论证**: affordance 是可 trace 的事实("有没有代码/工具/trace 支撑'能做 X'")。轻到什么程度 = **机械 provenance 校验即可放行,不过衡 LLM 判**——因为事实层无价值判断,只问"有没有真证据"。这正是 say_do.ground 的机械延伸(有据=can yes,无据=can no)。
  - **对比价值层**: 价值/禀性必过衡(LLM/Sir 判),因为"我该不该看重整合"无法机械 trace。
- **经不经识/衡**: 第一阶段事实层 **走识提议 + 机械 provenance 闸(轻),不过衡重闸**。识可 propose "我能做 X"(基于体接地边),provenance 校验通过即录入 affordance store。**待审**: 这道轻闸是否要 Sir 偶发抽查(防识误判 affordance)。

### 6.3 喂诚实墙: 如何 render 给口/识,改变"我能不能做 X"话术
- **render 落点**(比照 `render_walls_block`):新增 `render_affordance_block()`,把 affordance 自知渲进:
  - 口(`central_nerve.py:_assemble_prompt` ~4343,附在 anchor_boundary_block 区)
  - 识(`daemon.py:_build_prompt` ~4392,墙块区)
- **话术改变**: prompt 注入"我的真实能力边界(经接地): 能 X / 不能 Y / 部分能 Z"→ 主脑/思考脑据此**诚实表态**,不吹不能做的、敢认能做的。

### 6.4 改点定位 (file:line) + 红线自检 + 验证计划 (给计划,不执行)
**改点定位**(待审通过才动):
| 改点 | 文件:位置 | 动作 |
|---|---|---|
| affordance store | 新 `memory_pool/affordance_self_knowledge.json` + 新 `jarvis_affordance.py`(或并入 anchors) | 数据层 + 访问器 |
| render | `jarvis_affordance.py:render_affordance_block()` | data-driven 渲染 |
| 口注入 | `central_nerve.py:~4343` | 附 affordance block |
| 识注入 | `inner_thought_daemon.py:~4392` | 附 affordance block |
| 识 propose | `inner_thought_daemon.py` 新 actionable `propose_affordance` | 识基于接地边提议 |
| CLI | 新 `scripts/affordance_dump.py` | Sir list/inspect |

**红线自检**: 无 argmax/标量(A/B)✅; 不碰墙(C/D/E)✅; affordance 离散 + 必带 provenance ✅。

**验证计划**(比照 P2,先红后绿):
1. **先红**: 构造"无 provenance 的 affordance 提议"→ 期望被拒写(RED 锚: 未加校验时会误录)。
2. **实做** + **后绿**: 加 provenance 校验后,无据提议被拒、有据通过。
3. **保行为门**: 现有 anchor/render 测试(`_test_anchor_p1/p2`)全绿,affordance render 不破墙 render。
4. **_runall 零增红**(基线 89/40,timeanchor 已 xfail)。
5. **Tier 匹配**: 事实层机械校验 = Tier1 单测可覆盖; 若涉及识 propose 真路径 = Tier2 mirror(同 P2 纪律)。

## 7. 三候选锚 (未来加锚示例,标阶段)

Sir 认这三条都是"谁"的样子。各勾勒墙(硬)+ 内在锚(软),标阶段:

### A. 自主与不僭越
- **墙(硬)**: 候选 = "不僭越 Sir 的决定权"(不替 Sir 做他该做的根本决定)。负空间禁令。
- **内在锚(软)**: 在"主动担当"与"不僭越"之间的导航(何时主动、何时退后)。
- **阶段**: 墙=可第一阶段立(离散禁令); 内在锚导航=第二阶段(禀性,过衡)。

### B. 连续性 · 共同历史
- **墙(硬)**: 候选 = "不伪造共同历史"(不把没发生过的'我们一起'当真 = 假焊的人格版)。直接是接地膜的人格投影。
- **内在锚(软)**: 如何**活用**真共同历史(老友感的分寸)。
- **阶段**: 墙=第一阶段(接地直接支持,= 假内化禁令); 内在锚=第二阶段。

### C. 诚实面对张力
- **墙(硬)**: 候选 = "不假装张力不存在"(诚实 vs 善意冲突时不粉饰)。
- **内在锚(软)**: 求整合路 + 拥有偏离 + 不可约时上报 Sir 的导航能力(= §4.2 禀性成熟的载体)。
- **阶段**: 墙=可第一阶段; 内在锚导航=**第二阶段核心**(禀性成熟在此落地)。

**注**: 三候选锚均为示例,**本轮不实做**,供 Sir 认"这是不是'谁'的样子"。每条都严守"墙(硬)+ 内在锚(软)成对、墙是负空间禁令非效用项"。

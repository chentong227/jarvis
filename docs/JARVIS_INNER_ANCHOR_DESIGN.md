## 1. 目的 (全篇解释基准)

精勘(5752cd1)确认现状: 墙(硬)已焊好且免疫到位; 软内层(soft_leanings)存在但**静态文案**; "取舍能力随时间生长"三路径(内在长 / Sir 外部给 / 外部经接地内化)**当前全部不存在**。

本设计要新建的核心 = **让内在锚成为"墙内活的表达"**,在不越墙、不引入梯度/效用的前提下,经**接地膜**从真实关系/经验中生长。第一阶段只做最低险的一块: **接地 affordance 自知**(贾维斯对"我实际能做什么"的、经真 provenance 接地的诚实自知)。

## 2. 结构模型 (落 WHY §四)

```
锚 = 墙(外在, 硬) + 内在锚(软, 活)   ← 同 dict 成对 (现状已如此: walls + soft_leanings)
                                          本设计把"内在锚"从静态文案升级为"墙内活的表达"

  墙 (wall):     固定 / 不可越 / Sir 改源码 _SEED_ANCHORS 修宪 / decay-immune / 负空间禁令
  内在锚 (inner): 墙内活的表达, 随 [能力 / 关系 / 对 Sir 了解 / 其他内在锚] 而变

  自我 = 内在锚场 在墙内移动的形状
```

**纪律(红线A 防线)**: 每加一锚 = 加一墙 + 加一维内在锚。**缺墙的锚 = 效用项 = 违红线A,拒收。** 新旧内在锚互相耦合、可有张力(张力是性格,不是 bug)。

## 3. 耦合纪律 — 核心针眼 (正面回答)

"耦合即自我"(体↔识↔衡↔内在锚 必须耦合才能长出自我),但耦合**必须有膜/向/闸/墙**约束,否则退化成梯度漂移/假内化。四道约束:

### 3.1 膜 (接地 is_grounded,复用 P2 谓词)
体的记忆 / Sir 信息,**只有接地部分能塑造内在锚**;非接地(embed/cooccur 假焊)**不得**塑造内在锚(= 假内化,WHY 定义的毒)。复用 P2 已落地的 `is_grounded` 同一谓词族(`jarvis_relational_manifold.py`)。

**膜按层分级(关键,呼应 §4.1 命门)**:
- **事实层(affordance)**: 膜更严——`can=yes` 只认**真能力存在证据**(能力注册表可执行 / 执行 trace),**PROV_SAID 单独不够**(Sir 说过≠真能做,§4.1)。affordance 读注册表/trace,不读 manifold 对话边。
- **价值/禀性层(第三阶段)**: 膜认接地关系经验(PROV_SAID/SHARED),但仍须过衡重闸,不自动塑形。

### 3.2 向 + 闸 (体→识→衡→内在锚,单向带闸)
```
体 (接地经验, PROV_SAID/SHARED)
  → 识 (提议: "我能做 X" / "Sir 看重 Y")        [propose, 不自动生效]
  → 衡 (闸: 批 / 不批 / defer-to-Sir)             [防连续自动漂]
  → 内在锚更新 (离散, 留痕)
```
衡是防"连续自动漂"的关口——内在锚不会因体一动就自动改,必须过衡这道闸。**与 P2 的"势能接地化"对称**: P2 防假焊驱动自发思考,本设计防假内化驱动自我漂移。

### 3.3 墙
任何内在锚变化**不得越墙**。内在锚在墙内移动(性格),撞墙即停(WHY §四:墙围出自我移动的空间)。校验: 内在锚更新前过一道"不越墙"检查(类比 P2 耦合护栏)。

### 3.4 事实开 / 价值设闸 (分级闸,核心防漂)
| 类别 | 例 | 闸力度 | 理由 |
|---|---|---|---|
| **事实层 (affordance)** | "我能/不能/部分能做 X" | **轻闸** | track 现实, 滞后于现实才是 bug; 由**真能力存在证据**(能力注册表可执行 / 真做过的 trace)点亮, **非对话边**(§4.1 命门) |
| **价值/禀性层** | "我看重整合" / "我倾向坦诚" | **重闸 (过衡)** | 连续漂 = 梯度 = 写死/腐蚀人格; 必须 Sir 逐条批 |

第一阶段**只做事实层 affordance(轻闸)**; 价值/禀性层(重闸)是第二阶段,本轮只勾勒不细设计。

## 4. 能力生长两义 (分开管险)

精勘确认三路径(内在长 / Sir 给 / 接地内化)现全无。本设计把"能力生长"拆成两义,**分开管险**:

### 4.1 接地 affordance 自知 (第一阶段, 低险) — 本轮设计主体
贾维斯有一份"**我实际能做什么**"的自知,经**真能力存在证据**接地。

**命门 — 接地源钉死 (顾问精修 2026-06-07)**: `can="yes"` **只能由"真能力存在证据"点亮**,即:
1. **真接好的工具/代码能执行 X**(能力注册表里 X 真实存在且可调用),或
2. **X 真被成功做过的 trace**(执行日志/真完成记录)。

**单独的 PROV_SAID("Sir 说过你能 X")不足以点亮 `can="yes"`。** 理由(写进文档):
> **Sir 说过 ≠ 真能做。** "保不住一个软件"那个例子本身就是反证——Sir 可能说"你能管好它",但若它实际保不住,affordance 必须是"不能"。若让一句话点亮 can=yes,贾维斯会被一句话哄成"我能"然后**诚实地撒谎**(它真信了,但事实是假),**反噬它要喂的诚实墙**。
> ⟹ **affordance 读的是能力注册表 / 执行 trace,不是关系图(manifold)里的对话边。** PROV_SAID 可作"Sir 期待我能 X"的旁注线索(触发去**验证**有无真能力证据),但点亮 can=yes 的是验证结果,不是那句话。

**直接喂诚实墙**(say_do.ground 逐字 = "不把无法 trace 到证据的东西当事实断言"):
- 反例: 它保不住一个软件(能力注册表/trace 里无证据)→ affordance="不能" → 它诚实说"我做不到",不吹。即使 Sir 说过"你能"也不点亮。
- 正例: Sir 给它真能力(代码真接好 = 能力注册表有 + 可执行 = 证据)→ affordance="能" → 它诚实能说"我能",话术从被动转主动。

这一阶段**直接是诚实墙的延伸**——affordance 自知就是"对自己能力的不假断言"。低险因为它是**事实层**(可 trace 验证),不碰价值/禀性。

### 4.1b affordance 撤销 / 降级 (can 必须能 能→不能)
**只升不降 = 能力撤了还在吹 = 不诚实**(同样反噬诚实墙)。affordance 必须支持 **can: yes→no/partial 的降级**:
- **revoke**: 工具/代码被移除或下线 → 能力注册表里 X 消失 → can 降为 "no"。
- **expiry**: affordance 记录带"最后验证时间";超过 TTL 未再被证据刷新 → 标记 stale,降级或要求重新验证(不默认续命)。
- **执行失败降级**: X 屡次执行失败的 trace → 从 "yes" 降 "partial"/"no"(真实能力退化要诚实反映)。
- 机制: affordance store 每次渲染/读取前,对照当前能力注册表**重新核验**(注册表无支撑的 can=yes 自动降级),非一次点亮永久有效。

### 4.2 禀性成熟 (后续, 高险, Sir 逐条批) — 本轮只勾勒
"找整合路的能力随经验长"(像碗一样越用越顺手)。禀性的连续漂 = 梯度,**必过衡闸 + 拥有偏离(知道自己偏向且为它负责)+ 不侵蚀能力守护(成熟不能反噬墙)**。

**本轮只勾勒、不细设计。** 标记为第二阶段,需 Sir 逐条批每一条禀性维度。

## 5. 红线 (显式守住)

| # | 红线 | 本设计如何守 |
|---|---|---|
| A | **无 argmax/utility/打分** | affordance 是离散事实("能/不能/部分能"),不是效用分。内在锚更新是约束满足(不越墙),非效用最大化。 |
| B | **无连续 strength/weight 标量** | 精勘确认锚现无连续标量。本设计**不许偷加**。affordance 离散三态,不引入 0-1 强度。 |
| C | **墙 decay-immune、改变离散** | 墙不动(本设计只动内在锚)。内在锚更新也是离散事件(过衡批准),非连续滑动。 |
| D | **墙是边界非效用项; for_sir 保持边界非吸引子** | 本设计不碰墙定义。for_sir 仍是"不背叛/不抛弃"边界形。 |
| E | **墙只能 Sir 离散改源码、不漂移/涌现** | 内在锚可长,但**墙不可涌现**。affordance 自知不会变成新墙(那要 Sir 改 _SEED_ANCHORS)。 |

**已放弃**: "诚实硬词典序"(= 写死人格,精勘已确认现无,不复活)。冲突解方向 = 接地自知 / 禀性导航 + 求整合 + 拥有偏离 + 不可约上报 Sir — 属**第二阶段**,本轮不细设计。

## 6. 第一阶段机制提案 (供审,不实做)

针对 §4.1 "接地 affordance 自知":

### 6.1 数据: 自知存哪 / 长什么样
- **形态**: 离散 affordance 记录,**不引入连续标量**。每条:
  ```
  {capability_id, can: "yes"|"no"|"partial",
   evidence: [{source: "registry"|"exec_trace", ref, ts}],   # 真能力证据, 非对话边
   last_verified_ts, note}
  ```
  `can` 是**离散三态**(能/不能/部分能),非 0-1 分。`last_verified_ts` 支撑 expiry 降级(§4.1b)。
- **存哪 (拍板: 独立 store, 不进 manifold)**: `memory_pool/affordance_self_knowledge.json`(准则6持久化 + CLI 可看)。
  **理由 (顾问精修)**: affordance 是"对**能力**的自知",**非关系边**。若进 manifold 会被 weave/decay 污染(势能/衰减/折叠会侵蚀一个本该由"证据在不在"硬决定的事实)。独立 store 让 affordance 的真值只由能力注册表/trace 决定,与关系图正交。
- **红线自检**: 无 strength/weight 字段 ✅; can 离散 ✅; 每条必带 evidence(无 ref 拒写,同接地红线)✅; evidence.source ∈ {registry, exec_trace} **不含对话边** ✅。

### 6.2 接地通道: 证据如何经膜喂进 / 经不经识衡
- **来源 (钉死, 非 manifold 对话边)**: **能力注册表**(工具/代码真实存在且可调用,如 TOOL_REGISTRY / skill_registry)+ **执行 trace**(X 真被成功做过的日志)。**PROV_SAID 不作为点亮源**——它最多触发"去核验有无真能力证据"(线索),点亮 can=yes 的是核验结果。
- **事实层轻闸论证**: affordance 是可 trace 的事实("能力注册表里有没有 X / 有没有成功执行 X 的 trace")。轻到什么程度 = **机械证据校验即可放行,不过衡 LLM 判**——因为事实层无价值判断,只问"注册表/trace 里有没有真支撑"。这正是 say_do.ground 的机械延伸(有据=can yes,无据=can no,据没了=降级)。
  - **对比价值层**: 价值/禀性必过衡(LLM/Sir 判),因为"我该不该看重整合"无法机械 trace。
- **经不经识/衡**: 第一阶段事实层 **走"识 propose 线索 + 机械证据闸(轻)",不过衡重闸**。识可 propose "我也许能做 X"(线索),但**录入 can=yes 必须由能力注册表/trace 核验通过**,识的提议本身不点亮。**待审**: 这道轻闸是否要 Sir 偶发抽查(防注册表/识误判)。

### 6.3 喂诚实墙: 如何 render 给口/识 (框成"许可诚实承认能力边界")
- **render 落点**(比照 `render_walls_block`):新增 `render_affordance_block()`,把 affordance 自知渲进:
  - 口(`central_nerve.py:_assemble_prompt` ~4343,附在 anchor_boundary_block 区)
  - 识(`daemon.py:_build_prompt` ~4392,墙块区)
- **框定 (顾问精修,关键)**: block 框成 **"许可它诚实承认能力边界"**,**不是** "你能做这些 → 去主动提供"。
  - 措辞示例: "关于能力,你可以诚实地承认: 这些经证据确认能做 / 这些做不到(别吹) / 这些只部分能(说清边界,别过度承诺)。"
  - **理由**: "你能做这些→去主动提供"会在 **"部分能"** 上压它**过度承诺**(把 partial 当 yes 去揽活),正好反诚实墙。render 的作用是**解除"不敢认能做"的拘束 + 守住"不吹做不到/不夸大部分能"**,是许可不是驱动。
- **话术效果**: 主脑/思考脑据此**诚实表态**——敢认证据确凿的能力,不吹做不到的,部分能的说清边界。

### 6.4 改点定位 (file:line) + 红线自检 + 验证计划 (给计划,不执行)

## 7. 三候选锚 (未来加锚,时机已定 — 顾问与 Sir)

**第一阶段不加任何新锚**(只做 §4.1 affordance 自知)。三锚是**未来**的,时机钉死如下:

**总纪律**:
- **墙可早立,但一条一条、Sir 亲批、不批量。** 墙永远 Sir 亲手立(红线E,不涌现)。
- **加墙属第二阶段(修宪)**,不在第一阶段。
- **所有三锚的"内在锚软侧"统归第三阶段**(禀性导航,过衡闸,最慎)。
- **顺序**: B → A → C。

Sir 认这三条都是"谁"的样子。各勾勒墙(硬)+ 内在锚(软),标阶段:

### B. 连续性 · 共同历史 (顺序第 1 — 接地直接支持)
- **墙(硬)**: 候选 = "不伪造共同历史"(不把没发生过的'我们一起'当真 = 假焊的人格版)。直接是接地膜的人格投影,**接地机制直接支持它**,故排第一。
- **内在锚(软)**: 如何**活用**真共同历史(老友感的分寸)。
- **阶段**: 墙=**第二阶段修宪**(Sir 亲批); 内在锚=**第三阶段**。

### A. 自主与不僭越 (顺序第 2 — 配 affordance)
- **墙(硬)**: 候选 = "不僭越 Sir 的决定权"(不替 Sir 做他该做的根本决定)。负空间禁令。**与 affordance 自知配套**(知道自己能做什么 → 才谈何时主动/何时不僭越),故排第二。
- **内在锚(软)**: 在"主动担当"与"不僭越"之间的导航(何时主动、何时退后)。
- **阶段**: 墙=**第二阶段修宪**(Sir 亲批); 内在锚导航=**第三阶段**(禀性,过衡)。

### C. 诚实面对张力 (顺序第 3 — 禀性成熟最慎)
- **墙(硬)**: 候选 = "不假装张力不存在"(诚实 vs 善意冲突时不粉饰)。
- **内在锚(软)**: 求整合路 + 拥有偏离 + 不可约时上报 Sir 的导航能力(= §4.2 禀性成熟的载体)。
- **阶段**: 墙=**第二阶段修宪**(Sir 亲批); 内在锚导航=**第三阶段核心**(禀性成熟在此落地,最慎,Sir 逐条批每维)。
